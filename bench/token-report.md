## Honest limitation
This is the TOOL's OUTPUT token proxy — real for a consuming agent. It is NOT per-agent-session
token accounting (context + reasoning + tool I/O across a whole task), which is what the
BLUEPRINT's "small, high-signal context / fewer tokens" claim is really about. That needs runtime
telemetry from the agent host (e.g. Claude Code usage), which this repo cannot produce — so it is
stated, not invented. What is measured here is real; what isn't measurable in-repo is labelled so.
